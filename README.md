# NixOS · btrfs + Root-on-tmpfs + Preservation + Home Manager
** Complete [Phase 1](#2-phase-1--disk-setup) Download configs, place in /persist/etc/nixos, then skip to [Phase 4](#5-phase-4--install-nixos)**


A complete guide to installing NixOS with:
- **btrfs** filesystem with subvolumes
- **Root-on-tmpfs** — `/` lives in RAM and is wiped clean on every boot
- **Preservation** module — declarative whitelisting of what survives reboots
- **Home Manager** — user environment management via NixOS module
- A **modular flake** layout with persistence declarations in their own file

## 1. Architecture Overview

```
RAM (tmpfs)  ←  / (wiped on every boot — completely empty at startup)
             ↑
             └── Preservation module creates symlinks/bind-mounts at activation 
                       ↓
             /persist  (btrfs @persist subvolume — survives reboots)
             /nix      (btrfs @nix subvolume    — the Nix store, never wiped)
             /var/log  (btrfs @log subvolume    — optional persistent logs)
             /boot     (vfat EFI partition)
```

**What "root-on-tmpfs" means in practice:**
- On every boot, `/` is empty. Only files that are either generated by NixOS activation scripts (from the Nix store) or explicitly listed in `persist.nix` will exist.
- `/nix` and `/persist` are separate btrfs subvolumes mounted _before_ the Nix activation system runs, so they are always available.
- The Preservation module runs during activation and creates symlinks (for files) or bind-mounts (for directories) from their expected paths on tmpfs into `/persist`.

## 2. Phase 1 — Disk Setup

Boot into the NixOS live installer. All commands below assume your target disk is `/dev/sda`. **Adjust this** for your actual disk (e.g. `/dev/nvme0n1` for NVMe drives — partition names would then be `/dev/nvme0n1p1`, `/dev/nvme0n1p2`).

### 2.1 — Partition the disk

```bash
# Wipe any existing partition table
wipefs -a /dev/sda

# Create a GPT label, an EFI System Partition (512 MB), and a main btrfs partition
parted /dev/sda -- mklabel gpt
parted /dev/sda -- mkpart ESP fat32 1MB 512MB
parted /dev/sda -- set 1 esp on
parted /dev/sda -- mkpart primary 512MB 100%
```

### 2.2 — Format the partitions

```bash
mkfs.fat -F 32 -n BOOT /dev/sda1
mkfs.btrfs -L nixos /dev/sda2
```

### 2.3 — Create btrfs subvolumes

```bash
# Temporarily mount the btrfs top level to create subvolumes
mount /dev/sda2 /mnt

btrfs subvolume create /mnt/@nix     # The Nix store
btrfs subvolume create /mnt/@persist # All persisted state
btrfs subvolume create /mnt/@log     # System logs (optional but recommended)

umount /mnt
```

> **Why no `@root` subvolume?**
> Because we're using tmpfs for `/` — there's nothing to snapshot or roll back there. All the state that matters lives in `@persist`.

### 2.4 — Mount everything for installation

```bash
# 1. Mount tmpfs as the root — this is what / will be at runtime
mount -t tmpfs -o size=4G,mode=755 none /mnt

# 2. Create mountpoints
mkdir -p /mnt/{boot,nix,persist,var/log}

# 3. Mount the btrfs subvolumes
BTRFS_OPTS="compress=zstd,noatime"

mount -o "${BTRFS_OPTS},subvol=@nix"     /dev/sda2 /mnt/nix
mount -o "${BTRFS_OPTS},subvol=@persist" /dev/sda2 /mnt/persist
mount -o "${BTRFS_OPTS},subvol=@log"     /dev/sda2 /mnt/var/log

# 4. Mount the EFI partition
mount /dev/sda1 /mnt/boot
```

> **`size=4G` on tmpfs** limits how much RAM the root filesystem can consume at peak. Adjust to taste (e.g. `size=2G` for low-RAM machines or `size=50%` for a percentage of total RAM).

## 3. Phase 2 — Generate & Edit the Initial Config

### 3.1 — Generate hardware configuration

```bash
nixos-generate-config --root /mnt
```

This produces `/mnt/etc/nixos/hardware-configuration.nix`. However, because we want our flake at `/etc/nixos`, we need to move the generated files into `/persist` so they survive the first reboot:

```bash
mkdir -p /mnt/persist/etc/nixos
cp /mnt/etc/nixos/hardware-configuration.nix /mnt/persist/etc/nixos/
```

We'll write the rest of the config files directly into `/mnt/persist/etc/nixos/` in Phase 3.

### 3.2 — Edit hardware-configuration.nix

Open `/mnt/persist/etc/nixos/hardware-configuration.nix`. The auto-generated content will have the btrfs mounts. Verify it looks roughly like this and **add the `neededForBoot` flags and tmpfs root** if they weren't generated correctly:

```nix
# /etc/nixos/hardware-configuration.nix
{ config, lib, modulesPath, ... }:

{
  imports = [ (modulesPath + "/installer/scan/not-detected.nix") ];

  boot.initrd.availableKernelModules = [ "xhci_pci" "ahci" "nvme" "usb_storage" "sd_mod" ];
  boot.initrd.kernelModules = [ ];
  boot.kernelModules = [ "kvm-intel" ]; # or kvm-amd
  boot.extraModulePackages = [ ];

  # ── Root on tmpfs ──────────────────────────────────────────────────────────
  fileSystems."/" = {
    device  = "none";
    fsType  = "tmpfs";
    options = [ "defaults" "size=4G" "mode=755" ];
  };

  # ── EFI boot partition ─────────────────────────────────────────────────────
  fileSystems."/boot" = {
    device  = "/dev/disk/by-label/BOOT";
    fsType  = "vfat";
    options = [ "fmask=0077" "dmask=0077" ];
  };

  # ── Nix store — must be available early ───────────────────────────────────
  fileSystems."/nix" = {
    device          = "/dev/disk/by-label/nixos";
    fsType          = "btrfs";
    options         = [ "subvol=@nix" "compress=zstd" "noatime" ];
    neededForBoot   = true;
  };

  # ── Persistent state — must be available early (Preservation depends on it)
  fileSystems."/persist" = {
    device          = "/dev/disk/by-label/nixos";
    fsType          = "btrfs";
    options         = [ "subvol=@persist" "compress=zstd" "noatime" ];
    neededForBoot   = true;
  };

  # ── Persistent logs ────────────────────────────────────────────────────────
  fileSystems."/var/log" = {
    device          = "/dev/disk/by-label/nixos";
    fsType          = "btrfs";
    options         = [ "subvol=@log" "compress=zstd" "noatime" ];
    neededForBoot   = true;
  };

  swapDevices = [ ];

  nixpkgs.hostPlatform = lib.mkDefault "x86_64-linux";
  hardware.cpu.intel.updateMicrocode = lib.mkDefault config.hardware.enableRedistributableFirmware;
}
```

> **Important:** `neededForBoot = true` on `/persist` ensures it is mounted _before_ systemd tries to activate anything that depends on it. Without this you will get activation failures on a clean tmpfs boot.

## 4. Phase 3 — Build the Flake & Config Files

Your final `/etc/nixos` layout (stored at `/persist/etc/nixos` and symlinked by Preservation):

```
/etc/nixos/
├── flake.nix                    ← inputs, outputs, module wiring
├── flake.lock                   ← auto-generated, commit this to git
├── hardware-configuration.nix   ← auto-generated in Phase 2
├── configuration.nix            ← main system config; imports the others
├── persist.nix                  ← ALL preservation declarations live here
└── home.nix                     ← Home Manager user config
```

---

### flake.nix

```nix
# /etc/nixos/flake.nix
{
  description = "My NixOS configuration";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

    home-manager = {
      url = "github:nix-community/home-manager";
      inputs.nixpkgs.follows = "nixpkgs";
    };

    preservation = {
      url = "github:nix-community/preservation";
      # preservation doesn't pin nixpkgs itself, so no follows needed
    };
  };

  outputs = { self, nixpkgs, home-manager, preservation, ... } @ inputs:
  {
    nixosConfigurations."yourhostname" = nixpkgs.lib.nixosSystem {
      system = "x86_64-linux"; # change to aarch64-linux for ARM
      specialArgs = { inherit inputs; }; # makes `inputs` available in all modules
      modules = [
        # Core system config
        ./configuration.nix

        # Preservation NixOS module
        preservation.nixosModules.preservation

        # Home Manager as a NixOS module (single-user setup shown;
        # for multiple users, add more home-manager.users.<name> entries)
        home-manager.nixosModules.home-manager
        {
          home-manager = {
            useGlobalPkgs    = true;  # share nixpkgs from system config
            useUserPackages  = true;  # install user packages into /etc/profiles
            backupFileExtension = "bak"; # back up conflicting files rather than failing
            extraSpecialArgs = { inherit inputs; }; # passes `inputs` to home.nix
            users.yourusername = import ./home.nix;
          };
        }
      ];
    };
  };
}
```

> Replace `yourhostname` and `yourusername` throughout this guide.

### configuration.nix

```nix
# /etc/nixos/configuration.nix
{ config, pkgs, inputs, ... }:

{
  imports = [
    ./hardware-configuration.nix
    ./persist.nix           # ← persistence declarations live here, not inline
  ];

  # ── Bootloader ─────────────────────────────────────────────────────────────
  boot.loader.systemd-boot.enable      = true;
  boot.loader.efi.canTouchEfiVariables = true;

  # ── Networking ─────────────────────────────────────────────────────────────
  networking.hostName = "yourhostname";
  networking.networkmanager.enable = true;

  # ── Locale / Time ──────────────────────────────────────────────────────────
  time.timeZone               = "Australia/Melbourne"; # adjust to your timezone
  i18n.defaultLocale          = "en_AU.UTF-8";

  # ── Nix settings ───────────────────────────────────────────────────────────
  nix.settings = {
    experimental-features = [ "nix-command" "flakes" ];
    auto-optimise-store   = true;
  };

  # ── Users ──────────────────────────────────────────────────────────────────
  # Passwords are NOT set here as plaintext. Instead we use a hashed password
  # file stored on the persist subvolume. See Phase 4 for how to create it.
  users.mutableUsers = false; # IMPORTANT with tmpfs root: prevents /etc/passwd
                               # drift between reboots
  users.users.yourusername = {
    isNormalUser      = true;
    extraGroups       = [ "wheel" "networkmanager" "video" "audio" ];
    # Points to the hashed password file in the persist subvolume.
    # The file itself is created during installation (see Phase 4).
    hashedPasswordFile = "/persist/passwords/yourusername";
    shell             = pkgs.bash; # or pkgs.zsh, pkgs.fish, etc.
  };

  # Allow wheel group to use sudo
  security.sudo.wheelNeedsPassword = true;

  # ── System packages ────────────────────────────────────────────────────────
  environment.systemPackages = with pkgs; [
    git
    vim
    wget
    curl
    btop
    btrfs-progs
  ];

  # ── SSH ────────────────────────────────────────────────────────────────────
  services.openssh = {
    enable = true;
    settings = {
      PermitRootLogin    = "no";
      PasswordAuthentication = false;
    };
    # Store SSH host keys in persist (declared in persist.nix below)
  };

  # ── State version ──────────────────────────────────────────────────────────
  # Do NOT change this after first install. See `man configuration.nix`.
  system.stateVersion = "25.05";
}
```

### persist.nix

This is the heart of the setup. All preservation declarations live here. As your build grows, you add entries to this file without touching `configuration.nix`.

```nix
# /etc/nixos/persist.nix
#
# Preservation module configuration.
# All state that should survive reboots is declared here.
#
# How it works:
#   - Files:       a symlink is created at the expected path → /persist/<path>
#   - Directories: a bind-mount is created at the expected path → /persist/<path>
#
# The backing store for everything is the @persist btrfs subvolume,
# mounted at /persist.
#
# To add more entries later, just append to the `files` or `directories`
# lists and run `sudo nixos-rebuild switch`.

{ config, ... }:

{
  preservation = {
    enable = true;

    preservedAt."/persist" = {

      # ── Files ──────────────────────────────────────────────────────────────
      # Each entry is either a plain string (path) or an attrset with options.
      # Use attrsets when you need to control `how` (symlink/bindmount),
      # ownership (`user`, `group`), or permissions (`mode`).
      files = [

        # ── System identity ─────────────────────────────────────────────────
        # machine-id is used by systemd, journald, and D-Bus to identify this
        # host. If it changes, some services break or regenerate state.
        "/etc/machine-id"

        # Hardware clock drift adjustment (used by hwclock / timedatectl)
        "/etc/adjtime"

        # ── SSH host keys ────────────────────────────────────────────────────
        # These MUST persist. If they change, every SSH client will see a
        # "host key changed" warning and refuse to connect.
        { file = "/etc/ssh/ssh_host_rsa_key";     how = "symlink"; }
        { file = "/etc/ssh/ssh_host_rsa_key.pub"; how = "symlink"; }
        { file = "/etc/ssh/ssh_host_ed25519_key";     how = "symlink"; }
        { file = "/etc/ssh/ssh_host_ed25519_key.pub"; how = "symlink"; }

      ];

      # ── Directories ────────────────────────────────────────────────────────
      # Prefer `how = "bindmount"` for directories — it is more robust than
      # symlinking a whole directory, and some tools that check for real
      # directories (not symlinks) will work correctly.
      directories = [

        # ── NixOS state ──────────────────────────────────────────────────────
        # Stores user/group UID+GID allocations managed by NixOS.
        # WITHOUT this, NixOS will attempt to re-allocate IDs on every boot,
        # which can cause permission mismatches on files in /persist.
        {
          directory = "/var/lib/nixos";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0755";
        }

        # ── NixOS config itself ───────────────────────────────────────────────
        # Keeps your flake and all config files on the persist subvolume.
        # The symlink /etc/nixos → /persist/etc/nixos means `nixos-rebuild`
        # works as normal.
        {
          directory = "/etc/nixos";
          how  = "symlink";
          user = "root"; group = "root"; mode = "0755";
        }

        # ── NetworkManager ────────────────────────────────────────────────────
        # Saved Wi-Fi/VPN connection profiles. Without this you re-enter every
        # Wi-Fi password after each boot.
        {
          directory = "/etc/NetworkManager/system-connections";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0700";
        }

        # ── Bluetooth ─────────────────────────────────────────────────────────
        # Paired device database. Without this you re-pair devices every boot.
        {
          directory = "/var/lib/bluetooth";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0700";
        }

        # ── systemd coredumps (optional) ──────────────────────────────────────
        {
          directory = "/var/lib/systemd/coredump";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0755";
        }

        # ── sudo ──────────────────────────────────────────────────────────────
        # Stores the "sudo lecture seen" flag per user. Minor, but avoids
        # the sudo lecture on every first login after reboot.
        {
          directory = "/var/db/sudo/lectured";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0700";
        }

        # ── systemd timers state ──────────────────────────────────────────────
        # Preserves timer "last triggered" state so scheduled jobs don't all
        # fire simultaneously on the first boot after a long downtime.
        {
          directory = "/var/lib/systemd/timers";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0755";
        }

        # ── User home directories ─────────────────────────────────────────────
        # Persisting /home as a whole is the simplest approach:
        # Home Manager still manages dotfiles as store symlinks;
        # all user data (Documents, Downloads, browser profiles, etc.)
        # lives on the persist subvolume across reboots.
        {
          directory = "/home";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0755";
        }

        # ── Root home ─────────────────────────────────────────────────────────
        {
          directory = "/root";
          how  = "bindmount";
          user = "root"; group = "root"; mode = "0700";
        }

        # ════════════════════════════════════════════════════════════════════
        # Add your own entries below as your config grows.
        # Common additions:
        #
        # Docker / Podman:
        #   { directory = "/var/lib/docker";  how = "bindmount"; user = "root"; group = "root"; mode = "0710"; }
        #   { directory = "/var/lib/containers"; how = "bindmount"; user = "root"; group = "root"; mode = "0755"; }
        #
        # Printing (CUPS):
        #   { directory = "/var/lib/cups"; how = "bindmount"; user = "root"; group = "root"; mode = "0755"; }
        #
        # Pipewire / PulseAudio (per-user, handled better via HM preservation):
        #   { directory = "/var/lib/pipewire"; how = "bindmount"; user = "root"; group = "root"; mode = "0755"; }
        #
        # Tailscale VPN:
        #   { directory = "/var/lib/tailscale"; how = "bindmount"; user = "root"; group = "root"; mode = "0700"; }
        #
        # GNOME / GDM:
        #   { directory = "/var/lib/gdm"; how = "bindmount"; user = "gdm"; group = "gdm"; mode = "0711"; }
        #   { directory = "/var/lib/AccountsService"; how = "bindmount"; user = "root"; group = "root"; mode = "0775"; }
        #
        # Flatpak:
        #   { directory = "/var/lib/flatpak"; how = "bindmount"; user = "root"; group = "root"; mode = "0755"; }
        # ════════════════════════════════════════════════════════════════════

      ];
    };
  };
}
```

> **The "auto-whitelist files in `/persist`" question:** Files you manually place directly inside `/persist/` (e.g. `/persist/mydata/`) are automatically persistent — `@persist` is a real btrfs subvolume, not a symlink. You only need to declare entries in `persist.nix` for paths _outside_ `/persist` that should be transparently linked into the persist subvolume. Anything you deliberately place at `/persist/...` yourself is always safe.

## 5. Phase 4 — Install NixOS

### 5.1 — Set up the user password file

With `users.mutableUsers = false` and `hashedPasswordFile`, the password must exist on disk before the system can log in:

```bash
# Create the passwords directory on the persist subvolume
mkdir -p /mnt/persist/passwords
chmod 700 /mnt/persist/passwords

# Generate a bcrypt hash for your user password.
# You will be prompted to type a password interactively.
# `mkpasswd` is available in the NixOS installer.
mkpasswd -m sha-512 > /mnt/persist/passwords/yourusername
chmod 600 /mnt/persist/passwords/yourusername

# Do the same for root if desired (optional but recommended)
mkpasswd -m sha-512 > /mnt/persist/passwords/root
chmod 600 /mnt/persist/passwords/root
```

If you want a root password, add this to `configuration.nix`:
```nix
users.users.root.hashedPasswordFile = "/persist/passwords/root";
```

### 5.2 — Write your config files into persist

```bash
# All config files should already be in /mnt/persist/etc/nixos/
# (hardware-configuration.nix was copied there in Phase 2).
# Write the remaining files there now:

nano /mnt/persist/etc/nixos/flake.nix
nano /mnt/persist/etc/nixos/configuration.nix
nano /mnt/persist/etc/nixos/persist.nix
nano /mnt/persist/etc/nixos/home.nix
```

> **Tip:** You can also write your config on another machine and `scp` it in, or `git clone` your dotfiles repo directly into `/mnt/persist/etc/nixos/`.

### 5.3 — Pre-create directories that Preservation will bind-mount

The Preservation module creates bind-mounts at activation time, but it does NOT create the _backing_ directories in `/persist` for you on first boot if they don't exist. Create them now:

```bash
# Directories that Preservation will bind-mount from /persist
mkdir -p /mnt/persist/var/lib/nixos
mkdir -p /mnt/persist/var/lib/bluetooth
mkdir -p /mnt/persist/var/lib/systemd/{coredump,timers}
mkdir -p /mnt/persist/var/db/sudo/lectured
mkdir -p /mnt/persist/etc/NetworkManager/system-connections
mkdir -p /mnt/persist/home
mkdir -p /mnt/persist/root

# Set correct permissions
chmod 700 /mnt/persist/var/lib/bluetooth
chmod 700 /mnt/persist/root
chmod 700 /mnt/persist/var/db/sudo/lectured
chmod 700 /mnt/persist/etc/NetworkManager/system-connections

# Pre-create the nixos config directory symlink target
# (Already done — /mnt/persist/etc/nixos/ is where we wrote the config)
```

### 5.4 — Generate a machine-id and SSH host keys in persist

These are normally generated on first boot, but with tmpfs root they'd be lost without this:

```bash
# machine-id: a persistent 128-bit random identifier
systemd-machine-id-setup --root /mnt/persist --commit
# This writes /mnt/persist/etc/machine-id

# SSH host keys (Preservation will symlink /etc/ssh/* → /persist/etc/ssh/*)
mkdir -p /mnt/persist/etc/ssh
ssh-keygen -t rsa     -b 4096 -N "" -f /mnt/persist/etc/ssh/ssh_host_rsa_key
ssh-keygen -t ed25519         -N "" -f /mnt/persist/etc/ssh/ssh_host_ed25519_key
chmod 600 /mnt/persist/etc/ssh/ssh_host_rsa_key
chmod 600 /mnt/persist/etc/ssh/ssh_host_ed25519_key
```

### 5.5 — Install

```bash
nixos-install --root /mnt --flake /mnt/persist/etc/nixos#yourhostname --no-root-passwd
```

The `--no-root-passwd` flag skips the interactive root password prompt (we've already set it via `hashedPasswordFile`).

### 5.6 — Reboot

```bash
umount -R /mnt
reboot
```

## 6. First Boot Checks

After rebooting, verify the setup is working correctly:

```bash
# 1. Confirm / is a tmpfs (should show "tmpfs" as the filesystem type)
findmnt /

# 2. Confirm /persist and /nix are mounted btrfs subvolumes
findmnt /persist
findmnt /nix

# 3. Confirm /etc/nixos is a symlink to /persist/etc/nixos
ls -la /etc/nixos

# 4. Confirm machine-id is stable (symlink to /persist/etc/machine-id)
ls -la /etc/machine-id
cat /etc/machine-id

# 5. Confirm SSH host keys are symlinks
ls -la /etc/ssh/ssh_host_*

# 6. Check preservation activation succeeded
systemctl status preservation-*

# 7. Log in as your user and verify Home Manager is active
# (should show home-manager packages in your PATH)
which home-manager
```

From now on, to rebuild your system:

```bash
sudo nixos-rebuild switch --flake /etc/nixos#yourhostname
```

The `nrs` alias defined in `home.nix` does exactly this.

## 7. Adding More Entries to persist.nix Later

As you expand your OS build, the workflow is simply:

1. Edit `/etc/nixos/persist.nix`
2. Add an entry to `files` or `directories` under `preservedAt."/persist"`
3. **Create the backing directory in `/persist` first** (if it's a directory entry that doesn't exist yet):
   ```bash
   sudo mkdir -p /persist/var/lib/some-service
   sudo chown theuser:thegroup /persist/var/lib/some-service
   ```
4. Run `sudo nixos-rebuild switch --flake /etc/nixos#yourhostname`

The Preservation module's activation script will pick up the new entry and create the symlink or bind-mount. No reboot required.

## 8. Tips, Caveats & Troubleshooting

### tmpfs size
If you run out of space on the root tmpfs (e.g. large builds that write to `/tmp`), increase the `size=4G` option in `hardware-configuration.nix`. Alternatively, point `/tmp` to a btrfs subvolume instead of letting it live on the root tmpfs:
```nix
# In configuration.nix
boot.tmp.useTmpfs     = false;
boot.tmp.cleanOnBoot  = true;  # clears /tmp on boot (normal behaviour)
```

### The `/etc/nixos` symlink and `nixos-rebuild`
Because Preservation creates `/etc/nixos` as a _symlink_ to `/persist/etc/nixos`, `nixos-rebuild` works exactly as normal — it follows the symlink transparently. You can treat `/etc/nixos` as the canonical path.

### Git-tracking your config
Since `/etc/nixos` is on the persist subvolume, you can `git init` it and push to GitHub/GitLab:
```bash
cd /etc/nixos
git init
git add .
git commit -m "Initial NixOS config"
git remote add origin git@github.com:youruser/nixos-config.git
git push -u origin main
```
This also means `nixos-rebuild` will refuse to build untracked flake files — run `git add .` after creating new `.nix` files.

### Users and `mutableUsers = false`
With `users.mutableUsers = false`, you **cannot** use `passwd` to change passwords at runtime. To change a password:
```bash
# Generate new hash
mkpasswd -m sha-512 | sudo tee /persist/passwords/yourusername
sudo chmod 600 /persist/passwords/yourusername
# Then rebuild (the hash is read at activation, no rebuild needed in some cases,
# but a rebuild ensures everything is consistent)
sudo nixos-rebuild switch --flake /etc/nixos#yourhostname
```

### First boot: "activation script failed"
If the system fails to activate on first boot, the most common causes are:
- A Preservation backing directory in `/persist` that doesn't exist yet → SSH in via rescue mode and `mkdir -p /persist/...`
- Wrong permissions on `/persist` directories → check ownership with `ls -la /persist/...`
- The `neededForBoot = true` flag missing on `/persist` in `hardware-configuration.nix`

### Adding services that write to `/var/lib`
Most NixOS services store their state in `/var/lib/<service>`. With tmpfs root, this state is lost on reboot unless declared in `persist.nix`. The general pattern is:
```nix
{ directory = "/var/lib/service-name"; how = "bindmount"; user = "service-user"; group = "service-group"; mode = "0755"; }
```
Check `services.<name>.dataDir` in the NixOS options for the correct path; it's often documented in `man configuration.nix` or the [NixOS options search](https://search.nixos.org/options).

### Checking what's currently on tmpfs vs what's persisted
```bash
# See everything currently on the root tmpfs that isn't a mountpoint or symlink
find / -maxdepth 3 \
  -not -path '/proc/*' \
  -not -path '/sys/*' \
  -not -path '/nix/*' \
  -not -path '/persist/*' \
  -not -path '/run/*' \
  -not -path '/dev/*' \
  -not -type l \
  2>/dev/null
```
Anything that appears here but should survive reboots needs a `persist.nix` entry.
